1. Overview

Purpose

The Sporton Backend API provides a comprehensive e-commerce solution for sports equipment retail. It offers endpoints for managing products, categories, banking information, and customer transactions.

Base URL Local Development

http://localhost:5000/api

Base URL Local Development With DDEV

https://ikhfad-sporton-be.ddev.site/api

Base URL Production

https://ikhfad-sporton-be-production.up.railway.app/api

Content Types

The API supports multiple content types depending on the endpoint:

Content Type Usage Endpoints
application/json Standard JSON requests Auth, Banks, Transaction Updates
multipart/form-data File uploads with form fields Categories, Products, Transactions
application/x-www-form-urlencoded Simple form submissions All endpoints (fallback)

2. API Endpoints

🔒

Authentication

The Sporton API uses JSON Web Token (JWT) authentication. Write operations require a valid JWT token.

POST /api/auth/signin

POST /api/auth/signin application/json

Sign in to obtain a JWT token.

Request Body
{
  "email": "admin@example.com",
  "password": "your-secure-password"
}
Response 201 Created
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "507f1f77bcf86cd799439011",
    "name": "Admin User",
    "email": "admin@example.com"
  }
}
Using the Token
Authorization: Bearer YOUR_TOKEN_HERE
Token Expiration: Tokens expire after 24 hours.

POST /api/auth/initiate-admin-user

POST /api/auth/initiate-admin-user application/json

Create the initial admin user (only one admin allowed).

Request Body
{
  "name": "Admin Name",
  "email": "admin@example.com",
  "password": "secure-password"
}
Response 201 Created
{
  "message": "Admin user created successfully!"
}
